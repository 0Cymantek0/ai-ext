  <action>
    Optimize how the service worker broadcasts agent events to the side panel.

    File: ai-extension/src/background/service-worker.ts (or agent-runtime-service.ts if that's where events are emitted)

    Implementation:
    - Add an event buffer: Map<string, AgentRunEvent[]> keyed by runId
    - Add a flush timer per runId: Map<string, ReturnType<typeof setTimeout>>
    - When an event is emitted:
      1. Check if it's a critical event (approval.requested, run.completed, run.failed, run.cancelled)
      2. If critical: flush immediately — send chrome.runtime.sendMessage({ kind: "AGENT_RUN_EVENT", payload: { event } })
      3. If not critical: push to buffer for that runId
      4. If no flush timer exists for this runId, set one for 50ms
      5. When timer fires: flush all buffered events for that runId as individual messages, clear buffer and timer
    - Throttle high-frequency events:
      - Track last send time per event type per runId
      - If same event type sent within 100ms, skip (coalesce)
      - Exception: tool.called, tool.completed, tool.failed are always sent (but throttled to 10/second)
    - Clean up buffers and timers when run completes or is cancelled

    This reduces the number of chrome.runtime.sendMessage calls during active agent runs, preventing message queue saturation and side panel re-render storms.
  </action>
