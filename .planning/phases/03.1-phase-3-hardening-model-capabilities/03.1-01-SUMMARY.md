## Plan 03.1-01: Routing Types, Model Catalog & Settings Manager

### What was built

Extended the routing type system and built the data layer foundation for multi-provider routing:

1. **Extended types.ts** — Added `maxOutputTokens`, `supportsImageAnalysis`, `supportsVideoAnalysis`, `supportsAudioAnalysis` to `ModelCapabilities`. Added `providerId` and `enabled` to `ModelSheetEntry`. Added `CATALOG_VERSION` constant.

2. **Created model-catalog.ts** — Hardcoded capability overlay for 20+ models across OpenAI, Anthropic, Google, Gemini Nano providers. Includes `mergeWithOverlay` for unknown model defaults, `seedModelCatalog` for populating model sheets from enabled providers, and `MODEL_LIST_ENDPOINTS` for API-based model discovery.

3. **Extended settings-manager.ts** — Added 13 new methods with Zod and manual input validation:
   - Routing mode: `getRoutingMode`, `setRoutingMode` (Zod validation)
   - Fallback chain: `getFallbackChain`, `setFallbackChain` (provider validation)
   - Trigger words: `getTriggerWords`, `setTriggerWords`, `addTriggerWord`, `removeTriggerWord`
   - Model management: `addModel`, `removeModel`, `getModel`, `setModelEnabled`, `refreshModelCatalog`
   - Fixed shared constant mutation by returning fresh copies from `getModelSheet` and `getRoutingPreferences`.

### Key Decisions

- **Fresh copies for storage defaults**: `getModelSheet()` and `getRoutingPreferences()` return fresh objects instead of shared `DEFAULT_*` constants to prevent cross-test and cross-call mutation.
- **Zod for routing mode only**: `RoutingModeSchema` uses Zod with custom error map. Other methods use manual validation with specific error messages.
- **Provider validation in setFallbackChain**: Validates provider IDs against enabled providers via ProviderConfigManager.

### Tests

- **model-catalog.test.ts**: 12 tests covering HARDCODED_CAPABILITIES data, DEFAULT values, mergeWithOverlay, seedModelCatalog, and MODEL_LIST_ENDPOINTS.
- **settings-manager.test.ts**: 26 tests (5 original + 21 new) covering all new methods with validation error checks.
