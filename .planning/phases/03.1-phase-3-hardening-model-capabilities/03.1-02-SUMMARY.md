## Plan 03.1-02: ProviderRouter Enabled Filtering & Comprehensive Tests

### What was built

Added enabled filtering to ProviderRouter at both provider and model levels, with comprehensive test coverage for all routing paths:

1. **Provider-level enabled filtering** — `routeCapability()` checks `config.enabled` before attempting adapter creation. Disabled providers are skipped with diagnostic logging.

2. **Model-level enabled filtering** — Execution chain checks that at least one enabled model exists for the provider (via `modelSheet` entries where `entry.providerId === providerId && entry.enabled !== false`).

3. **Heuristic scoring with enabled filtering** — Auto mode scoring loop skips disabled models (`entry.enabled === false`) and providers with `enabled === false` before computing scores.

4. **Actionable error messages** — When all providers are filtered out, throws `"No enabled providers available for '{capability}' capability"` with diagnostic details.

5. **Model sheet keyed by modelId** — Updated heuristic scoring variable from `providerId` to `modelKey` and uses `entry.providerId` for bestMatch routing.

### Key Decisions

- **ConfigManager initialized early** — Moved `getProviderConfigManager()` initialization before auto mode block so it's available for both heuristic scoring and execution chain filtering.
- **Model sheet fetched early** — `getModelSheet()` called once at the start, shared between heuristic scoring and enabled filtering.
- **Error diagnostics include skip reasons** — Each skipped provider records reason (disabled, no models, not found).

### Tests

- **provider-router.test.ts**: 14 tests across 7 describe blocks covering:
  - Primary provider success, fallback chain iteration
  - Auto mode heuristic scoring with nano classifier integration
  - Disabled primary provider → fallback routing
  - All providers disabled → actionable error
  - Provider with no enabled models → skipped
  - Auto mode disabled model/provider filtering
  - Embeddings and speech capability routing
  - Trigger word matching and fallback to heuristics
  - Fallback chain with mixed enabled/disabled providers
